volatile переменные видно из всех потоков!
поэтому если много чтений из разных потоков, то вместо того, чтобы синхронизировать метод get(),
можно просто юзать volatile переменную.
Т.е. volatile - это lock-free механизм! потому он и быстрее (чем синхронизация через мониторы)

ВНИМАНИЕ: volatile НЕ спасает в случаях, когда
1) значение переменной зависит от ее предыд значения (т.е. нельзя юзать для счетчиков)
2) когда переменная входит в инвариант (например, volatile int minimum, int maximum (если min <= max) нельзя!)
